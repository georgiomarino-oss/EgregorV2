## Asset generation pipeline

Source files:

1. `mobile/scripts/generate-cinematic-assets.mjs`
2. `mobile/assets/generated/asset-prompts.json`
3. `mobile/assets/generated/asset-manifest.json`

Run commands:

```powershell
# Optional: provide key for OpenAI image generation
$env:OPENAI_API_KEY="your_key_here"

# Generate or refresh the full cinematic asset set
npm --prefix mobile run assets:generate:cinematic

# Dry-run validation
npm --prefix mobile run assets:generate:cinematic:dry
```

Behavior:

1. If `OPENAI_API_KEY` is present, the script attempts OpenAI Images API generation.
2. If key is missing (or provider call fails and strict mode is not used), the script generates deterministic procedural fallback assets so app surfaces still render with premium media instead of placeholder boxes.
3. Script always updates `asset-manifest.json` with usage mapping, provider used, and file checksums.
